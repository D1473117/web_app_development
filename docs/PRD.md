## 1. 專案概述

### 1.1 背景與動機

現代人生活忙碌，常常在網路上看到喜歡的食譜卻沒有統一的地方儲存，最終散落在不同的書籤或手抄紙上，找起來十分不便。「食譜收藏夾」旨在提供一個集中管理食譜的平台，讓使用者能夠輕鬆新增、整理、搜尋自己喜愛的料理食譜。

### 1.2 目標用戶

| 角色 | 描述 |
|------|------|
| **一般用戶** | 對烹飪感興趣的一般大眾，希望能儲存與搜尋食譜 |
| **管理員** | 負責維護平台內容，具備審核、編輯與刪除食譜的管理權限 |

### 1.3 核心價值主張

- 📌 **集中管理**：將所有食譜儲存在同一平台，方便查閱
- 🔍 **靈活搜尋**：支援關鍵字與食材組合兩種搜尋方式
- 🏷️ **分類整理**：透過標籤與分類幫助使用者快速找到目標食譜
- 👨‍🍳 **社群分享**：用戶可以公開食譜與他人交流料理心得

## 2. 功能需求

### 2.1 功能列表與使用者故事

#### 功能一：使用者帳號管理

> 作為**一般用戶**，我希望能夠**註冊帳號並登入系統**，以便**儲存個人食譜收藏並保護隱私**。

- 使用者可以用 Email + 密碼進行註冊與登入
- 支援登出功能
- 管理員帳號由系統預設設定

---

#### 功能二：新增與管理食譜

> 作為**一般用戶**，我希望能夠**新增、編輯與刪除自己的食譜**，以便**維護個人的食譜收藏庫**。

- 食譜欄位包含：名稱、描述、食材清單、步驟、烹調時間、難易度、分類、封面圖片
- 食譜可設為「公開」或「私人」
- 只有建立者本人（與管理員）可以編輯或刪除

---

#### 功能三：瀏覽食譜

> 作為**一般用戶**，我希望能夠**瀏覽平台上公開的食譜列表**，以便**發現新的料理靈感**。

- 首頁顯示最新/熱門公開食譜
- 食譜卡片顯示：封面圖、名稱、作者、分類、難易度
- 支援分頁或無限捲動
- 可依分類（如：湯品、主餐、甜點）篩選

---

#### 功能四：關鍵字搜尋食譜

> 作為**一般用戶**，我希望能夠**使用關鍵字搜尋食譜**，以便**快速找到特定料理**。

- 可依食譜名稱、描述進行模糊搜尋
- 可組合篩選條件（分類、難易度、烹調時間）
- 搜尋結果顯示相關度排序

---

#### 功能五：依食材搜尋食譜

> 作為**一般用戶**，我希望能夠**輸入手邊有的食材來搜尋對應食譜**，以便**利用現有食材做出料理，減少食物浪費**。

- 使用者可以輸入多種食材（標籤輸入方式）
- 系統回傳「包含所有指定食材」或「包含部分食材」的食譜
- 食材匹配度高的食譜排列在前

---

#### 功能六：收藏食譜（我的最愛）

> 作為**一般用戶**，我希望能夠**將喜歡的食譜加入我的收藏**，以便**之後快速找回想做的料理**。

- 已登入用戶可以按愛心收藏任何公開食譜
- 個人頁面顯示「我的收藏」清單
- 可取消收藏

---

#### 功能七：留言與評分

> 作為**一般用戶**，我希望能夠**對食譜留言並打星評分**，以便**分享料理心得並幫助他人判斷食譜品質**。

- 每則食譜下方有留言區
- 評分為 1～5 星，平均分數顯示在食譜頁面
- 只有登入用戶才能留言與評分

---

#### 功能八：管理員後台

> 作為**管理員**，我希望能夠**審核、編輯與刪除任何食譜或留言**，以便**維持平台內容品質**。

- 管理員可查看所有食譜（含私人）
- 管理員可刪除違規食譜與留言
- 管理員可查看用戶清單並停用帳號

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| **後端框架** | Flask（Python） |
| **模板引擎** | Jinja2 |
| **資料庫** | SQLite |
| **前端** | HTML5 + CSS3 + 原生 JavaScript |
| **部署環境** | 本機開發 / 學校伺服器 |

### 3.2 效能考量

- 食譜列表頁面的查詢回應時間應在 **1 秒內**
- 圖片上傳大小限制為 **5MB 以內**
- 搜尋功能應支援至少 **500 筆食譜**不明顯變慢

### 3.3 安全考量

- 密碼使用 **bcrypt** 或 Flask 內建方式加密儲存，不得明碼儲存
- 使用 **Session** 管理登入狀態
- 表單需防止 **CSRF 攻擊**（使用 Flask-WTF 或手動加 token）
- 防止 **SQL Injection**（使用 ORM 或參數化查詢）
- 上傳的圖片檔案需驗證副檔名與 MIME Type
